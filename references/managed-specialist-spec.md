# Managed Specialist Spec

Managed specialists are runtime-installed artifacts derived from canonical state
in `.expert-team/`.

## Required folder shape

Each installed specialist should have:

```text
<managed-name>/
├── SKILL.md
└── agents/openai.yaml
```

## Required behavior

Every managed specialist must:

- be derived from exactly one `template_id`
- be specialized by the current project dossier
- be narrowed by approved memory only
- stay concise and role-specific
- point back to canonical project state rather than duplicating the full dossier
- be installed into the active runtime's chosen scope rather than a hardcoded global path

## Required `SKILL.md` frontmatter

- `name`: installed managed skill name such as `fantasypulse-growth-lead`
- `description`: concise statement of role, project specificity, and when to use it

## Required `SKILL.md` body sections

- `Overview`
- `Mandate`
- `Owns`
- `Does Not Own`
- `Expected Deliverables`
- `Success Metrics`
- `Read These References First`
- `Operating Rules`

Optional section:

- `External Data Use`, when capability is not `local-only`

## Required managed marker

Each managed specialist should include a short machine-readable marker near the
top of `SKILL.md`, for example:

```text
Managed by: expert-team-bootstrapper
Project slug: fantasypulse
Role id: growth-lead
Template id: growth-lead
Source of truth: .expert-team/team-spec.json
Install scope: project
```

This marker makes reconciliation safer and prevents accidental modification of
unrelated user-created skills.

## Required `agents/openai.yaml` fields

- `interface.display_name`
- `interface.short_description`
- `interface.default_prompt`

The default prompt should use the installed managed skill name.

## Update rules

On update:

- overwrite managed specialists generated by this bootstrapper
- preserve the managed marker and provenance fields
- do not edit unrelated skills even if their names are similar

On removal:

- remove only specialists that are marked as managed by this bootstrapper and
  no longer appear in the current manifest or team spec
